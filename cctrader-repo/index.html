What it does

Once loaded, a floating panel appears in the top-left showing five paper-traded strategies plus the live bot. The live bot (Sim-EV ★) uses the game-accurate forecast to decide buys and sells every tick. The others are benchmarks — they show you that the forecast is actually earning its keep versus simpler rules.

Panel header has an Auto-trade: ON/OFF toggle — flip off to keep forecasting and paper-trading without spending cookies. Rows in the in-game market panel get tinted green when the bot would buy, red when it would sell.

How the forecast works

Cookie Clicker's stock market rules are fully documented on the wiki. Each tick, every stock is updated by a known set of operations involving delta decay, reversion to resting value, random fluctuations, and mode-specific effects. The trader reads the current (value, delta, mode, modeAge) of every stock directly from the game state, then runs 200 forward simulations 40 ticks ahead using these exact rules.

Mean endpoint price becomes the forecast μ. Standard deviation becomes σ. The trader then computes:

const buyEV  = forecast_mean − price × (1 + overhead) − λ × σ;
const sellEV = price − forecast_mean − λ × σ;

Decisions are thresholded on these:

ActionConditionDefault threshold
BUYbuyEV > minBuyEdge AND room to buy$0.10/unit
SELLsellEV > minSellEdge AND hold any$0.10/unit
HOLDneither edge clears—

Default λ = 0 — risk-neutral, only expected value matters. Bump it up if you want the bot to chicken out on uncertain trades.

Console API

Everything is exposed via window.CCTrader:

// Control
CCTrader.start()                  // resume the tick hook
CCTrader.stop()                   // pause
CCTrader.uninstall()              // remove entirely
CCTrader.runOnce()                // manually trigger one tick

// Diagnostics
CCTrader.status()                 // current config + stats summary
CCTrader.explain('Farm')          // full decision breakdown for one stock
CCTrader.benchmarks()             // per-strategy P&L table
CCTrader.portfolioBreakdown()     // current positions + unrealized P&L
CCTrader.tradeLog()               // last 200 executed trades

// Tuning
CCTrader.setConfig({ horizon: 40, riskAversion: 0.0 })

Known limitations

  • Supreme Intellect: decay rate and mode duration are aura-adjusted correctly. Global spike rate, widened fluctuations, and the 50% Chaotic-override on standard mode transitions are NOT yet modeled. Decisions are reasonable with SI equipped, but forecast σ is slightly understated.
  • Warehouse upgrades: the bot doesn't try to level up your Bank or buy Stockbrokers. Do that yourself — brokers especially make a big difference (overhead drops from 20% to 1% at 59 brokers).
  • DOM integration for row colors: uses guessed element ID patterns. If the game updates its market UI, the colors may stop appearing. CCTrader.setConfig({ colorizeStockRows: false }) disables cleanly.

Why

Because Cookie Clicker has a 10-year-old documented stochastic process underlying its stock market that nobody seems to have pointed a proper filter at. The game tells you the current mode. It exposes the hidden delta. The per-mode dynamics are in a table on the wiki. At that point not building this feels rude.

It started as an AR(1) approximation over a hidden Markov model and got progressively more accurate as I dug further into the source. The current version simulates the game's exact rules forward 40 ticks, 200 times per stock, every minute. Forecasts take ~10 ms each; a full tick of 15 stocks is ~400 ms.
